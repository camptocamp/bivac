Install from source

$ go get github.com/camptocamp/bivac

Example usage

$ bivac \
  -u s3+http://s3-eu-west-1.amazonaws.com/<my_bucket>/<my_dir> \
  --aws-access-key-id=<my_key_id> \
  --aws-secret-key-id=<my_secret_key>

Launch with Docker

$ docker run -v /var/run/docker.sock:/var/run/docker.sock:ro  --rm -ti \
   -e BIVAC_TARGET_URL=s3+http://s3-eu-west-1.amazonaws.com/<my_bucket>/<my_dir> \
   -e AWS_ACCESS_KEY_ID=<my_key_id> \
   -e AWS_SECRET_ACCESS_KEY=<my_secret_key> \
     camptocamp/bivac

Features

Bivac allows to backup Container volumes using the Duplicity, RClone, and Restic engines.

Docker and Kubernetes Support

Bivac plugs onto the Docker socket or the Kubernetes API to detect volumes and back them up, automatically.

Prometheus Ready

Prometheus Support is built-in to easily ship metrics each time bivac runs.

Multiple Clouds

Bivac supports both AWS and Swift as bucket providers.

Multiple Engines

Bivac uses Duplicity, RClone and Restic as backends to give you full control of your backup mode.
